The Benefits of HXL

The Humanitarian Exchange Language

The Humanitarian Exchange Language (HXL) is a simple standard for messy data. Use HXL hashtags to speed up data processing and create interoperability across data sources.

HXL was developed for humanitarians, by humanitarians. It takes into account the specific challenges humanitarians face in collecting, processing, and sharing data as well as the tools that humanitarians actually use to do their work.

While that is all well and good, you are probably asking yourself "what will HXL actually help me do? Why should I care about HXL?" In this section, we will tell you why you should care about HXL.

Benefits of HXL

The HXL data standard was designed to make working with messy humanitarian data more efficient and effective. As a humanitarian, HXL can help if:

  • you feel bogged down by repetitive, manual workflows;
  • you are looking for ways to reduce errors when preparing, cleaning, and merging data;
  • you want to quickly and easily be able to know what data is in a given dataset;
  • you have ever dreamed about producing reusable dashboards and quickly whipping up new visualisations; and
  • you put your blood, sweat and tears into producing high quality data and you want to make sure other people can use it.

Automate Data Workflows

Humanitarians spend an enormous amount of time and efforts manually copying, preparing, validating, and cleaning data. HXL was designed to transform repetitive, manual workflows to ensure that humanitarians can spend their time getting to insight.

Reduce Errors in Data

It is easy to lose confidence in humanitarian data when it is riddled with many minor errors. We know that these errors are the result of overworked humanitarians who just do not have the time to spot them all.

Improve Data Clarity

Between different languages and naming conventions that are found throughout the sector, it can be difficult to know what the data in a humanitarian dataset HXL is allows data users to know what data is in a dataset without having to know every language or search through data dictionaries.

Reproducible Dashboards and Visualisations

Humanitarians know that there is a lot of duplicated efforts building similar visualisations and dashboards over and over. HXL and tools it powers can help humanitarians produce reusable dashboards and quickly and easily create new visualisations.

Help Downstream Users Reuse your Data

We spend an enormous amount of time collecting and preparing data and share it in the hopes that other humanitarians can benefit. Data standards like HXL help downstream users quickly and easily get what they need from our data without requiring us to redesign how we work. Its a win-win!

Hashtags & Attributes

HXL Hashtags

The root of a HXL hashtag follows the same syntactic rules as a Twitter hashtag: it begins with the pound sign (“#”) and contains only unaccented Roman alphabetic characters (“a” to “z”), Arabic numerals (“0” to “9”), and the underscore symbol (“_”).

The core shared HXL hashtags describe high-level concepts like an organisation (#org), geographical coordinates (#geo), a humanitarian cluster or sector (#sector), the number of people affected (#affected), or a subdivision of a country (#adm1).

#inneed
#population

HXL Attributes

The attributes of the HXL hashtag are used to specify characteristics. They do not carry complete information on their own and can only be understood when combined with the hashtag they are describing.

HXL attributes start with “+” sign and always follow the hashtag that they describe. It is HXL compliant to add a space before the "+", but not after it. A hashtag may have any number of attributes, and order does not matter, so #org+funder+code has exactly the same meaning as #org+code+funder.

+start
+name

Attributes Are Reuseable

HXL attributes are not tied to a single HXL hashtag. Think of them as adjectives that add descriptive details to the hashtag they describe.

The attribute "+children" can be combined with hashtags to describe the following unique variables:

  • #population +children: Number of children in the general population.
  • #affected +children: Number of children affected by the emergency.
  • #inneed +children: Number of children in need of humanitarian assistance.
  • #reached +children: Number of children reached with humanitarian assistance.

Where to Add HXL Tags

HXL works with your existing spreadsheets and other humanitarian data.

  1. Add a new row between the last row of headers and the first row of data values.
  2. In your new row, add a HXL hashtag for each column you want to label.
  3. Use HXL attributes to refine the meaning of the hashtags.
  4. You do not need to add HXL tags to every column in the spreadsheet.

Core Hashtags

The core HXL hashtags can be grouped into five topic areas:

  1. Places: These include hashtags like #country, #adm3 and #geo. They are used when you have data on the administrative level or a geolocation.
  2. People and households: These include hashtags like #affected and #inneed. They are used when you have data about the people affected by the crisis and their needs
  3. Responses and other operations: These include hashtags like #org and #indicator. They are used when you have data about the response by organisations and people seeking to help those who need assistance.
  4. Crises, incidents and events: These include hashtags like #impact and #crisis. and
  5. General metadata: These include hashtags like #date and #status.

Let's take a look at the place and people and households hashtags in more detail.

Place Hashtags

Hashtag Description Suggested Attributes
#region Supra- or cross-national region (e.g. Sahel, Horn of Africa, Caribbean). +name, +code, +type
#country Country (often left implied in a dataset). +name, +code, language attributes like +l_ar
#adm1 Top-level subnational administrative area (e.g. a governorate in Syria). +name, +code, language attributes like +en
#adm2 Second-level subnational administrative area (e.g. a municipio in Colombia). +name, +code, language attributes like +ru
#loc Any point location, such as a village, camp, or clinic. +name, +code, +type, language attributes like +id
#geo Geodetic geometry information (points, lines, shapes). Use for latitude and longitude, as well as bounds information. +lat, +lon, +elevation, +border, +bounds, +url (to link to external boundary/shape data, generally in GeoJSON format).

People Hashtags

Hashtag Description Suggested Attributes
#affected Number of people or households affected by an emergency. +num, +type, SADD attributes like +f, impact attributes like +idps, +label
#beneficiary Beneficiary information. +num, +type, SADD attributes like +elderly, impact attributes like +displaced, +label
#inneed Number of people or households in need of assistance. +num, +type, SADD attributes like +children, impact attributes like +injured, +label
#population Population characteristics. +num, +type, SADD attributes like +m, impact attributes like +infected, +label
#reached Number of people or households reached with assistance. +num, +type, SADD attributes like +infants, impact attributes like +noncamp, +label
#targeted Number of people or households targeted for assistance +num, +type, SADD attributes like +adolescents, impact attributes like +incamp, +label

Refining Hashtags using Attributes

Core HXL hashtags like #inneed specify the thing a column of data is talking about (people in need of humanitarian assistance). However, we often need to be able refine this hashtag to make it more precise (is the data about male or female people in need? adults or children? refugees or IDPs? in camp or self-hosted?).

Data providers may invent their own attributes to suit their local data needs. However, there are some recommended common attributes that will be useful across many data types. We will go over how to use the following to refine the meaning of a hashtag:

  1. Sex and Age Dissaggregated Attributes
  2. Language Attributes
  3. Impact Att
  4. Name, Code and Classifiers

Sex and Age Attributes

Sex and age disaggregated data (SADD) is an important part of many humanitarian datasets. The following HXL attributes can be used with sex and age disaggredated data. It's important to note that age categories, in particular, tend to vary from country to country. HXL age attributes do not try to capture the precise differences but rather will provide a loose level of interoperability (e.g. are we talking about younger people or adults?).

HXL Hashtag Description
+f applies to female persons
+m applies to male persons
+i applies to persons who don’t identify as male or female
+infant applies to very young children
+children applies to children (in general)
+adolescents applies to adolescents
+adults applies to adults (in general)
+elderly applies to elderly people

Impact Attributes

These attributes refine tags like #affected, #reached or #inneed to specify different kinds of impacts that a crisis has had on the population specified.

HXL Attribute Description
+idps those displaced internally.
+infected those infected with disease as a result of the crisis.
+killed those killed as a result of the crisis.
+refugees those displaced to other regions or countries.
+displaced those displaced from their homes by the crisis.
+incamp those displaced into official camps.
+injured those physically injured (but not killed) as a result of the crisis.
+noncamp those displaced people hosted outside of official camps.

Language Specific Tags

Some humanitarian datasets combine information in multiple languages (e.g. Pashto and Dari in Afghanistan). The HXL standard suggests reserving all two-character attributes for use as language codes, e.g. +ar for Arabic, +es for Spanish, +en for English, and +fr for French. You can then use those attributes to disambiguate columns with equivalent text in different languages, e.g. #description+fa for Dari and #description+ps for Pashto.

HXL Tag Description Data Value
#country +name +i_fr Name in French République centrafricaine
#country +name +i_sg Name in Sango Ködörösêse tî Bêafrîka
#country +name +i_en Name in English Central African Republic
#country +name +i_ar Name in Arabic جمهورية أفريقيا الوسطى
#country +name +i_ru Name in Russian Центральноафриканская Республика

Name, Code & Type

In a our data, we have different ways of describing the same thing. We may refer to a country by its name or by an ISO-3 code. We may describe an organisation by its name or by the type of organisaiton it is.

In HXL, we deal with this by using the following common attributes:

  • +name is used when our column contains data for the name of something. This is what humans call the thing.
  • +code is used when our column contains data for the standarised code of something. This is what computers might call the thing.
  • +type is used when our column contains data on how to classify things.

Now lets take a look at each of these attributes in a bit more detail.

Name Attribute

The attribute +name is used when a column contains a human-readable name. This tag can be combined with the language specific tags to specify what language the name is in. Let's look at a few examples of the name attribute being used.

HXL Tag Description Data Value
#country +name Country name Koulikouro
#adm1 +name Admin level-1 subdivision name Kati
#adm2 +name Admin level-2 subdivision name Ténenkou
#loc +camp +name Camp name Baguineda Camp
#sector +cluster +name Cluster name Health
#crisis +name Crisis name Sahel Crisis

Code Attribute

The attribute +code is used when a column contains a unique machine readable standardised code such as an ISO-3 number or P-code. Let's look at a few examples of the name attribute being used.

HXL Tag Description Data Value
#country +code M49/ISO3 country code MLI
#adm1 +code Admin level-1 P-code ML02
#adm2 +code AAdmin level-2 P-code ML0205
#loc +camp +code P-code for a camp ML020501
#sector +cluster +code OCHA code for a cluster HEA
#crisis +code GLIDE code for a crisis SOT-2011-000205-NER

Type Attribute

The attribute +type is used when a column contains a classifier. The attribute +code if the classifier is from a code list. Let's look at a few examples of the name attribute being used.

HXL Tag Description Data Value
#loc +type Type of location Camp
#crisis +type Type of crisis Drought
#event +type Type of event attack
#org +type Type of organisation iNGO
#value +type Type of financial label Budget
#item +type Type of item Food

HXL Rules and Conventions

Column Order Does Not Matter

HXL tags label the column in your spreadsheet, which means that you can change the column order (or header names) without wreaking havoc in downstream workflows. In practice, this means that you can rearrange, remove or add extra columns to your spreadsheet. HXL-aware software will simply skip the columns it is not able to read.

This flexibility makes sharing data in spreadsheets more error resistant.

HXL Tags Separate Headers from Data

While HXL uses hashtags to label columns of data, it still allows you to include human-readable headers. The headers go before the row containing the hashtags. The data starts in the next row. You can include up to 24 rows of headers before the HXL hashtag row.

You Can Change Header Names

Just like with column order, making changes to the headers will not break any HXL-based processing as long as the HXL hashtags stay the same. This makes it possible to make your data interoperable without changing your existing workflows or data schemas while allowing you to retain the flexibility to change header names if you need to.

Use Multiple Columns for Data with Multiple Values

Humanitarian data often contains more than one value for the same type of information. For example, an aid project might target two countries or involve two different organisations. Create your Own Tags

The HXL core tags include tags that will be generally applicable to many humanitarian datasets, but it is impossible to anticipate every tag for every humanitarian need. If you do not find what you need in the core HXL hashtag and attribute dictionary, you can create your own hashtags starting with “x_”, such as #x_virulence or #x_risk. This standard makes the following four recommendations for extending HXL tags and attributes:

  • Whenever possible, take an existing tag with a broader meaning, and narrow it down with an attribute, e.g. #loc+hospital.
  • When there is no applicable core tag, begin an extension tag with “x_”, so that it will not conflict with any future HXL core hashtags, e.g. #x_toxicity.
  • When software finds a HXL hashtag that it does not recognise, e.g. #x_toxicity, it should simply ignore the column of data.
  • When software finds a HXL attribute that it does not recognise, e.g. #loc+hospital, it should ignore the attribute but still process the tag and any other attributes it does recognise: in this case, it should behave as if the dataset had contained simply #loc.
